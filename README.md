# Генератор Vanity Адресов BEP20

Скрипт для генерации кошельков BEP20 (Binance Smart Chain) с адресами, соответствующими заданным маскам.

## Описание

Программа генерирует приватные ключи и соответствующие адреса BEP20 до тех пор, пока не найдет адрес, который соответствует заданным маскам:
- Первые 2 символа после `0x`
- Последние 4 символа

## Требования

- C++20 компилятор (GCC, Clang)
- CMake 3.10 или выше
- OpenSSL библиотека

### Установка зависимостей

**macOS:**
```bash
brew install openssl cmake
```

**Ubuntu/Debian:**
```bash
sudo apt-get install libssl-dev cmake build-essential
```

## Компиляция

```bash
mkdir -p build
cd build
cmake ..
make
```

## Использование

```bash
./CryptoSpider
```

Программа запросит:
1. Первые 2 символа после `0x` (можно использовать `?` для любого символа)
2. Последние 4 символа (можно использовать `?` для любого символа)

### Примеры масок:

- Префикс: `ab`, Суффикс: `1234` - адрес должен начинаться с `0xab...` и заканчиваться на `...1234`
- Префикс: `??`, Суффикс: `0000` - адрес должен заканчиваться на `...0000` (первые 2 символа любые)
- Префикс: `00`, Суффикс: `????` - адрес должен начинаться с `0x00...` (последние 4 символа любые)

## Выходные данные

Программа выводит:
- Найденный адрес BEP20
- Приватный ключ в hex формате
- Количество попыток
- Время поиска
- Скорость генерации адресов

## ⚠️ ВАЖНО

- **Сохраните приватный ключ в безопасном месте!**
- **Никому не показывайте приватный ключ!**
- **Приватный ключ дает полный доступ к кошельку!**
- Чем больше символов в маске, тем дольше будет поиск (экспоненциально)

## Как использовать приватный ключ

Приватный ключ можно импортировать в любой кошелек, поддерживающий BEP20:
- MetaMask
- Trust Wallet
- Binance Chain Wallet
- И другие EVM-совместимые кошельки

## Производительность

Программа оптимизирована для максимальной скорости:
- **Многопоточность**: использует все доступные ядра CPU
- **Оптимизированные алгоритмы**: работа с байтами вместо строк
- **Переиспользование контекстов**: минимизация выделения памяти

Скорость генерации зависит от:
- Количества ядер процессора (линейное масштабирование)
- Сложности маски (количество фиксированных символов)
- Производительности процессора

**Типичная скорость**: 
- На 4-ядерном процессоре: 50,000 - 200,000 адресов/сек
- На 8-ядерном процессоре: 100,000 - 400,000 адресов/сек
- На 16-ядерном процессоре: 200,000 - 800,000 адресов/сек

*Скорость может варьироваться в зависимости от конкретного процессора и сложности маски*

## Примечания

- Адреса BEP20 используют тот же формат, что и Ethereum адреса
- Программа использует криптографически стойкий генератор случайных чисел
- Keccak-256 реализация соответствует стандарту Ethereum
